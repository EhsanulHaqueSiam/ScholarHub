**4. Create `web/convex/related.ts` with scoring algorithm (D-68 through D-78):**

Export `DEFAULT_RELATED_WEIGHTS = { provider: 35, country: 25, degree: 15, funding: 15, tags: 10 }`.

Export `scoreRelated(source, candidate, weights)` pure function that computes weighted score:
- Provider match (exact string match on provider_organization): weights.provider if match
- Country match (exact string match on host_country): weights.country if match
- Degree overlap (proportional): `intersectionSize(source.degree_levels, candidate.degree_levels) / Math.max(source.degree_levels.length, 1) * weights.degree`
- Funding match (exact match on funding_type): weights.funding if match
- Tag overlap (proportional): if source has tags, `intersectionSize(sourceTags, candidateTags) / sourceTags.length * weights.tags`

Helper `intersectionSize(a: string[], b: string[]): number` counts common elements.

Export `computeRelatedIds(ctx, doc, weights?)` async function that:
1. Gets weights from `related_weights` table if exists, else uses DEFAULT_RELATED_WEIGHTS
2. Queries published scholarships with `.take(50)` (bounded, per anti-pattern warning)
3. Filters out expired (application_deadline < Date.now()), the source doc itself, and archived
4. Scores each candidate using scoreRelated
5. Sorts by score descending, takes top 6 (D-73: 4-6 cards)
6. Returns array of scholarship IDs

Export Convex query `getRelatedScholarships` that takes `scholarshipId` and returns the related scholarship docs by reading `related_ids` from the scholarship and fetching each.

Export Convex query `getRelatedWeights` for admin to read current weights.
Export Convex mutation `updateRelatedWeights` for admin to update weights (D-69).
