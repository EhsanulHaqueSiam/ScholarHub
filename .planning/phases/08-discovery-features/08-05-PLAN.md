**3. Create `web/src/routes/collections/$slug.tsx` (D-44, D-11):**

Route: `/collections/$slug` with `createFileRoute`.

Page title: "{Collection Name} | ScholarHub" (D-113).

Content:
- Navbar at top
- CollectionHeader component
- Sort pills (reuse SortPills component from directory). Default sort from collection's `default_sort` field. Student can override (D-11).
- ViewToggle (grid/list)
- ScholarshipCard or ScholarshipListItem grid (reuse from directory) based on view toggle state
- Pagination (reuse DesktopPagination from directory) -- offset-based since getCollectionScholarships returns total count

State:
- `sort`: from collection.default_sort, overridable via SortPills
- `view`: "grid" | "list", default "grid"
- `page`: number, default 1
- Page size: 20

Queries:
- `useQuery(api.collections.getCollectionBySlug, { slug: params.slug })` for collection metadata
- `useQuery(api.collections.getCollectionScholarships, { slug: params.slug, sort, limit: 20, offset: (page-1)*20 })` for scholarships

View counter: `useMutation(api.collections.recordCollectionView)` called once via useEffect with localStorage debounce key `collection_view_{slug}`. Only fire if key doesn't exist or was set > 30 minutes ago (D-89).

Loading: CollectionHeader skeleton (circle + heading lines + badge) + 6 SkeletonCard grid (D-108).
Error: "This collection could not be loaded. It may have been archived or removed." Button: "Browse All Collections" linking to /collections (per copywriting contract).
404 case: if getCollectionBySlug returns null, show error state.
