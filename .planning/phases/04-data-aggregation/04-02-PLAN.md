       Then call `.paginate(...)`.
    2. For each raw_record in `results.page`:
       a. Skip if missing title or provider_organization (set match_status="new", patch canonical_id to null to skip next time -- actually just skip these, leave them for manual review).

       c. Look up existing scholarship by match_key using `ctx.db.query("scholarships").withIndex("by_match_key", q => q.eq("match_key", matchKey)).collect()` to get all candidates.

       d. **Full match (D-01):** Filter candidates where `hasDegreeLevelOverlap(candidate.degree_levels, record.degree_levels ?? [])` is true.
          - If exactly one full match found: **MERGE** (see merge logic below).
          - If multiple full matches: pick the one with status "published" or most recent _creationTime. Merge into that one.

       e. **Partial match (D-03):** If no full match, check if any candidate exists with same match_key but NO degree overlap. If so, flag raw_record with `match_status: "possible_duplicate"` and do NOT merge. Create a new scholarship entry.

       f. **No match:** Create new canonical scholarship entry.

    3. **Merge logic** (when full match found) per D-05/D-06/D-07:
       - Load the source document for this raw_record: `ctx.db.get(record.source_id)`.

       - Patch the scholarship with merged field values.

       - Set raw_record.canonical_id to the existing scholarship ID.
       - Set raw_record.match_status to "matched".

       **For fields_of_study merge:** Union all fields from all sources.
       **For funding booleans (funding_tuition, funding_living, etc.):** OR — if any source says true, result is true.
