## Interaction Contract

### Sticky Bar Visibility
- **Trigger:** IntersectionObserver on hero section ref
- **Show when:** Hero section is NOT intersecting viewport (threshold 0)
- **Animation:** `transition-transform duration-200` with `-translate-y-full` toggle
- **Contains:** Truncated title + Copy Link (neutral button) + Apply Now (default button)

### Copy Link
- **Pattern:** Reuse from ScholarshipCard.tsx
- **Behavior:** `navigator.clipboard.writeText(url)` on click
- **Feedback:** Button text changes from "Copy Link" to "Link copied!" with Check icon for 2000ms
- **SSR guard:** URL constructed inside onClick handler only (no window access during render)

### Apply Now
- **Element:** `<a>` styled as Button, `target="_blank"` `rel="noopener noreferrer"`
- **URL:** `scholarship.application_url`
- **Fallback when no URL:** Button disabled with text "Application Link Unavailable"
- **Appears in:** Hero section (full width on mobile, auto width on desktop) AND sticky bar (size sm)

### Nationality Expand/Collapse
- **Default:** Show first 10 nationalities as flag emoji + country name badges
- **Trigger:** Click "+N more countries" text button
- **Expanded:** Full list grouped by region (Africa, Americas, Asia, Europe, Middle East, Oceania, Other)
- **Region headers:** font-heading text-sm, followed by comma-separated country names with flags
- **Collapse:** Click "Show fewer" text button to return to first 10
- **State:** Local useState, no URL persistence needed

### "Open to All" Nationalities
- **Condition:** `eligibility_nationalities` is an empty array `[]`
- **Display:** Prominent banner with globe icon: "Open to All Nationalities" heading + "Students from any country are eligible to apply" body
- **Style:** `bg-urgency-open/10 border-2 border-urgency-open rounded-base p-4`

### Missing Nationality Data
- **Condition:** `eligibility_nationalities` is `undefined` or `null`
- **Display:** Muted text: "Eligibility information not available. Check the official application page for details."

### Deadline Countdown (hydration-safe)
- **SSR render:** Date only, no countdown (to avoid hydration mismatch)
- **Client mount:** useEffect calculates daysLeft, updates every 60 seconds
- **Display format:** "March 31, 2026 (12 days left)" -- date is always visible, countdown appears after hydration
- **Timezone:** Show in user's local timezone with abbreviation: "Your time: March 31, 2026 at 5:59 AM (UTC+6)"
- **No fabricated scholarship timezone:** Only user's detected timezone is displayed

### Expired Deadline
- **Display:** "Applications Closed" badge (urgencyClosed variant)
- **Reopen hint:** If `expected_reopen_month` exists, show "Expected to reopen: September 2026"
- **Apply button:** Disabled with text "Applications Closed"

### Breadcrumb Navigation
- **Format:** "Scholarships > {Scholarship Title}"
- **"Scholarships" link:** Points to `/scholarships` with preserved filter params from `?ref=` search params
- **Title:** Truncated with `truncate max-w-[160px] md:max-w-[300px]` as final non-link segment
- **Separator:** `>` character in `text-foreground/40`

### Stale Data Warning
- **Condition:** `last_verified` timestamp is >30 days old
- **Display:** Amber highlight box inside SourcesSection: "This information may be outdated. Last verified over 30 days ago."
- **Style:** `bg-urgency-warning/10 border border-urgency-warning rounded-base p-3 text-sm`

### Editorial Tips Callout
- **Present:** Callout box with `bg-main/5 border-2 border-main rounded-base p-4`
- **Icon:** Lightbulb from lucide-react, size-5, positioned left of heading
- **Heading:** "ScholarHub Tips" in font-heading text-sm
- **Body:** Rendered via react-markdown with allowed elements: p, strong, em, a, ul, ol, li
- **Link styling:** Links in editorial notes get `text-main underline underline-offset-4`
- **Absent:** Show muted placeholder "Tips coming soon -- we're working on adding insider advice for this scholarship." in `text-foreground/50 text-sm italic`

### Loading State (DetailSkeleton)
- **Layout:** Matches section card structure -- 6 skeleton cards stacked
- **Hero skeleton:** Large title bar (h-8 w-3/4) + provider bar (h-5 w-1/2) + badge row (3 small bars)
- **Section skeletons:** Title bar (h-6 w-1/3) + 3-4 content bars of varying widths
- **Animation:** `motion-safe:animate-pulse` (matches existing SkeletonCard pattern)
- **Wrapper:** Same max-w-3xl mx-auto container as real content

### Not Found State
- **Reuse:** Existing 404 pattern from placeholder but enhanced
- **Heading:** "Scholarship Not Found" in font-heading text-[32px]
- **Body:** "The scholarship you're looking for doesn't exist or has been removed."
- **CTA:** "Browse All Scholarships" link button to `/scholarships`

## SEO Contract

### Meta Title Format
`{Title} -- {Funding Type} {Degree} Scholarship in {Country} | ScholarHub`

Example: `DAAD Scholarship -- Fully Funded PhD Scholarship in Germany | ScholarHub`

Fallback (missing data): `{Title} | ScholarHub`

### Meta Description
First 155 characters of `scholarship.description`. Fallback: `Learn about {Title} from {Provider} -- eligibility, funding, deadlines, and how to apply. | ScholarHub`

### Schema.org JSON-LD (expanded from placeholder)
Expand existing `buildScholarshipJsonLd` to include:
- `@type`: "Scholarship" (custom, project convention)
- `name`: scholarship title
- `description`: scholarship description
- `provider`: { @type: Organization, name: provider_organization }
- `url`: application_url or detail page URL
- `applicationDeadline`: ISO 8601 date string
- `eligibleRegion`: array of { @type: Place, name: country_name } for each nationality
- `studyLocation`: { @type: Place, name: host_country_name }
- `amount`: { @type: MonetaryAmount, value: award_amount_max, currency: award_currency }
- **New fields to add:**
  - `educationalLevel`: degree_levels array (e.g., ["PhD", "Masters"])
  - `occupationalCategory`: fields_of_study array
  - `funder`: { @type: Organization, name: provider_organization }
  - `dateModified`: last_verified ISO date (if available)

## Dark Mode Contract

All section cards, sticky bar, and hero use CSS custom properties that automatically adapt via `.dark` class toggle (already established in `index.css`). No component-level dark mode logic needed.

Specific dark mode considerations for this phase:
- Editorial tips callout: `bg-main/5` adapts naturally since `--main` is consistent across modes
- Stale warning: `bg-urgency-warning/10` adapts via `--urgency-warning` dark mode value
- "Open to All" banner: `bg-urgency-open/10` adapts via `--urgency-open` dark mode value
- Funding checklist icons: use token-based colors (`text-urgency-open`, `text-urgency-closed`, `text-foreground/40`) that adapt automatically
- Skeleton pulse: `bg-border/20` adapts via `--border` dark mode value (already established pattern)

## Accessibility Contract

- All section cards use semantic HTML: `<section>` with `aria-labelledby` pointing to section heading `id`
- Breadcrumb uses `<nav aria-label="Breadcrumb">` with `<ol>` list
- Funding checklist icons have `aria-label`: "Covered", "Not covered", "Not specified"
- Apply Now link: `aria-label="Apply to {scholarship title} (opens in new tab)"`
- Copy Link: `aria-label` toggles between "Copy link to {title}" and "Link copied"
- Sticky bar: `role="banner"` with `aria-label="Quick actions"`
- Nationality expand button: `aria-expanded` attribute toggles
- External links: `rel="noopener noreferrer"` on all `target="_blank"` links
- Focus visible: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-ring` (project pattern)
- Reduced motion: `motion-safe:animate-pulse` for skeleton, `motion-safe:transition-transform` for sticky bar
- Color contrast: All urgency badge variants maintain WCAG AA contrast (white text on colored backgrounds)
