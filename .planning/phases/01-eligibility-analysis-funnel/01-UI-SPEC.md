## Interaction Contracts

### Wizard Step Transitions (D-04)
- **Direction**: Forward = slide left (translateX -100%), Back = slide right (translateX +100%)
- **Duration**: 300ms
- **Easing**: ease-out (CSS `transition: transform 300ms ease-out`)
- **Implementation**: CSS transitions only, no framer-motion. Container has `overflow-hidden`. Exiting step slides out while entering step slides in simultaneously
- **Reduced motion**: Respect `prefers-reduced-motion` via Tailwind `motion-safe:` prefix. If reduced motion preferred, instant swap with no animation

### Step Bar (D-05)
- **Layout**: Horizontal bar at top, full width. Three segments: "About You" | "Academics" | "Preferences"
- **States**: Inactive (border-border bg-transparent text-foreground/50), Active (border-border bg-main text-main-foreground), Completed (border-border bg-accent text-accent-foreground + checkmark icon)
- **Connector line**: 2px solid border-border between steps. Completed segments fill with bg-main
- **Clickable**: Completed steps are clickable (D-06). Active and future steps are not clickable. Cursor pointer on completed, default on others
- **Mobile**: Step bar scrolls away with page content. Labels abbreviated if needed ("About" | "Academics" | "Prefs")

### Live Match Count (D-07)
- **Position**: Centered below step bar, above form content
- **Format**: "{count} scholarships match so far"
- **Styling**: font-heading text-xl, count number in accent color background badge (bg-accent text-accent-foreground border-2 border-border px-3 py-1)
- **Animation**: Count number transitions with a brief scale pulse (scale 1 -> 1.1 -> 1, 200ms) when value changes. Use CSS `transition: transform 200ms ease-out`
- **Loading**: Show "Checking..." with pulse animation while Convex query is in flight
- **Zero state**: "No matches yet -- keep filling in your details"

### Form Inputs
- **Text/number inputs**: border-[3px] border-border bg-secondary-background h-14 px-4 text-base font-base shadow-shadow focus:translate-x-boxShadowX focus:translate-y-boxShadowY focus:shadow-none (matches existing SearchBar pattern)
- **Select/dropdown**: Same border treatment. Use Radix Select with neo-brutalism styling
- **Multi-select (searchable)**: Popover with search input at top, scrollable checkbox list. Selected items shown as badges below input
- **Radio groups**: Custom styled like existing checkbox pattern (neo-brutalism appearance override)
- **Required field indicator**: Asterisk (*) in font-heading text-destructive after label text
- **Optional field label**: "(optional)" in text-foreground/50 text-[13px] after label text
- **Validation errors**: text-destructive text-[13px] font-base below the field, border-destructive on the input

### Navigation Buttons (D-06)
- **Next button**: Button variant="default" size="lg" -- right-aligned, text "Next: Academics" / "Next: Preferences" / "Find My Scholarships"
- **Back button**: Button variant="neutral" size="lg" -- left-aligned, text "Back"
- **Mobile (D-36)**: Both buttons in a sticky footer bar at bottom of viewport. bg-secondary-background border-t-2 border-border px-4 py-3. Back on left, Next on right. Full-width if only one button
- **Desktop**: Buttons at bottom of wizard card, flex justify-between

### Results Page Layout (D-22, D-23)
- **Profile summary card (D-26)**: Full width at top, Card component. Displays nationality flag + name, degree, fields, destinations as inline badges. Each badge is clickable to edit inline (opens small popover with the field editor). Background: bg-secondary-background
- **Tier sections (D-23)**: Stacked vertically. Each has a colored header bar (bg-match-{tier}-bg border-2 border-match-{tier}-border) with tier label + count. Body contains scholarship cards in a responsive grid (1 col mobile, 2 col md, 3 col lg). Sections are collapsible -- click header to toggle
- **Default state**: All tier sections expanded. Possible matches section collapsed by default if other tiers have results
- **Mobile (D-37)**: Full-width stacked cards. Tier headers become sticky as user scrolls past them (position: sticky, top: 64px for navbar offset). z-index: 40

### Match Indicators on Cards (D-24)
- **Position**: Bottom of CardContent, before CardFooter
- **Layout**: Horizontal row of 4 small indicators: Nationality | Degree | Field | Language
- **Each indicator**: 24px icon (Check/X/Minus from lucide) + 13px label below. Green check = match, red X = no match, gray dash = unknown
- **Expandable**: Click the indicator row to expand a full match breakdown panel below the card. Panel shows each dimension with explanation text
- **Compact by default**: Only icons visible, no labels. Labels appear on hover (desktop) or in expanded view

### Editable Profile Summary (D-26)
- **Click to edit**: Each profile field in the summary card is wrapped in a button-like container. On click, a small inline popover opens with the same input component from the wizard step
- **Live update**: When user changes a value in the popover and closes it, results re-query and re-score immediately
- **Visual treatment**: Editable fields have a subtle dashed bottom border (border-dashed border-b border-foreground/20) and a small pencil icon (Pencil from lucide, size-3) on hover

### Empty State (D-27)
- **Reuse pattern**: Follow existing EmptyState component pattern with neo-brutalism SVG illustration
- **Content**: Heading + body text explaining which criteria are restrictive + suggestion to relax + link to directory
- **"Browse all scholarships" link (D-28)**: Button variant="neutral" size="lg" that links to /scholarships with nationality and degree pre-filled as URL params

### Welcome Back Flow (D-30)
- **Detection**: Check localStorage for existing profile on /eligibility route mount
- **UI**: Card centered on page with: "Welcome back! We found your previous profile." + profile summary preview + two buttons: "View My Results" (variant="default") and "Update Profile" (variant="neutral")
- **Skip wizard**: "View My Results" navigates directly to /eligibility/results with stored profile params

### Start Over (D-31)
- **Button**: variant="destructive" size="sm", text "Start Over", positioned in profile summary card footer
- **Confirmation**: Browser confirm() dialog: "This will clear your saved profile and start fresh. Continue?"
- **Action**: Clear localStorage, navigate to /eligibility

## Accessibility Contracts

- All form inputs have associated `<label>` elements with `htmlFor`
- Required fields marked with `aria-required="true"` and visual asterisk
- Step bar uses `role="list"` with `role="listitem"` for each step, `aria-current="step"` on active
- Wizard uses `role="form"` with `aria-label="Eligibility wizard - Step {n} of 3"`
- Live match count uses `aria-live="polite"` for screen reader announcements
- Tier sections use `<details>` / `<summary>` or `aria-expanded` + `aria-controls` pattern
- Match indicators have `aria-label` text: "Nationality: matched" / "Degree: not matched" / "Field: unknown"
- Keyboard navigation: Tab through form fields, Enter to submit step, Escape to close popovers
- Focus management: When sliding to next step, focus moves to first input of new step
- Touch targets: All interactive elements have minimum 44px touch target (min-h-[44px])
- Color contrast: All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Neo-brutalism's high-contrast borders ensure this
