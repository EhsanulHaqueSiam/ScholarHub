       - Use module-level cache variables: `let interFont: ArrayBuffer | null = null; let archivoFont: ArrayBuffer | null = null;`
       - On first request, fetch from Google Fonts CSS API, extract the TTF/OTF URL (NOT WOFF2 -- satori does not support WOFF2), fetch the font file as ArrayBuffer
       - Google Fonts TTF URL pattern: `https://fonts.googleapis.com/css2?family=Inter:wght@400&text=a` returns CSS with a `url(...)` pointing to the font file. Parse with regex.
       - Alternative simpler approach: use the direct Google Fonts API v2 endpoint which returns font URLs. OR bundle a fallback with hardcoded Google Fonts CDN URLs for Inter and Archivo Black TTF files.
       - Actually, the simplest reliable approach: use the Google Fonts Developer API or known CDN URLs. Inter TTF is available at `https://fonts.gstatic.com/s/inter/v18/UcCO3FwrK3iLTeHuS_nVMrMxCp50SjIw2boKoduKmMEVuLyfAZ9hiA.woff2` -- but this is WOFF2. For TTF, use a different source or the npm package `@fontsource/inter`.

       Best approach for reliability: Bundle the font files. Include Inter-Regular.ttf and ArchivoBlack-Regular.ttf in `web/src/routes/api/` or a `web/src/assets/fonts/` directory. Read them with `fs.readFileSync` or dynamic import.

       Actually, in a Netlify Function context, file system access is limited. The most reliable approach:
