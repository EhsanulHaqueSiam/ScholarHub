**File 1: `.github/workflows/ci.yml`** (runs on PRs to main)

```yaml
name: CI

on:
  pull_request:
    branches: [main]

jobs:
  lint-and-typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2

      - name: Install web dependencies
        run: cd web && bun install --frozen-lockfile

      - name: Lint (Biome)
        run: cd web && bun biome check .

      - name: Type check
        run: cd web && bun tsc --noEmit
```

**File 2: `.github/workflows/deploy.yml`** (runs on push to main)

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2

      - name: Install web dependencies
        run: cd web && bun install --frozen-lockfile

      - name: Lint (Biome)
        run: cd web && bun biome check .

      - name: Type check
        run: cd web && bun tsc --noEmit

      - name: Deploy to Netlify
        run: cd web && npx convex deploy --cmd 'bun run build'
        env:
          CONVEX_DEPLOY_KEY: ${{ secrets.CONVEX_DEPLOY_KEY }}
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
```

**Important notes on the deploy workflow:**
- `CONVEX_DEPLOY_KEY` is set in GitHub repo secrets (from Convex Dashboard > Settings > Generate Production Deploy Key)
- `NETLIFY_AUTH_TOKEN` and `NETLIFY_SITE_ID` are set in GitHub repo secrets (from Netlify Dashboard)
- The `npx convex deploy --cmd 'bun run build'` command handles both Convex function deployment and Vite build in one step
- Uses `oven-sh/setup-bun@v2` for Bun runtime in CI
- Uses `bun install --frozen-lockfile` to ensure reproducible installs from bun.lockb
- Python/scraping CI is deferred to Phase 3 per user decision

Also add `web/package.json` scripts if not already present:
