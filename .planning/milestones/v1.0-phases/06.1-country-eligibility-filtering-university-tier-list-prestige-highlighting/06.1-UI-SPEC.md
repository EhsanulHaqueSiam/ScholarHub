> Visual and interaction contract for the public scholarship directory with country eligibility filtering, prestige tier system, and neo-brutalism design. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in badges, inline pill padding |
| sm | 8px | Card internal gaps between badge rows, filter chip gaps |
| md | 16px | Card padding (px-4), filter bar internal padding, grid gap on mobile |
| lg | 24px | Card vertical padding (py-6 already on Card), section heading margin-bottom |
| xl | 32px | Gap between filter sections, grid gap on desktop (gap-8) |
| 2xl | 48px | Hero section vertical padding, major section margin |
| 3xl | 64px | Page top/bottom margin, hero to directory gap |

Exceptions:
- Touch targets: 44px minimum height for all tappable elements (filter chips, sort pills, country selector items, mobile bottom sheet triggers). This ensures WCAG 2.5.5 compliance.
- Neo-brutalism shadow offset: 4px (already defined as `--box-shadow-x` / `--box-shadow-y`)

## Layout Contract

### Page Hierarchy (Desktop, top to bottom)

1. **Navbar** -- fixed top, full width, 56px height
2. **Hero Section** -- full width, 2xl vertical padding (48px). Display heading + subtitle + SearchBar centered
3. **Smart Eligibility Filter Bar** -- full width, below hero. When compact: single line with edit button
4. **Directory Section** -- max-width 1280px, centered
   - Quick filter tabs (row)
   - Active filter chips (row, wrapping)
   - Sort pills + View toggle + Results count (row, space-between)
   - Content area: FilterPanel sidebar (280px, left) + Results grid/list (flex-1, right)
5. **Load More / Pagination** -- centered below results
6. **Back to Top** -- floating, bottom-right, 16px from edge

### Page Hierarchy (Mobile, top to bottom)

1. **Navbar** -- fixed top, full width, 48px height
2. **Hero Section** -- full width, condensed (lg padding, 24px). Display heading smaller (24px). SearchBar full width.
3. **Smart Eligibility Filter Bar** -- full width, always visible
4. **"Filters" button** -- sticky below eligibility bar, opens bottom sheet (Radix Dialog as sheet)
5. **Quick filter tabs** -- horizontal scroll row
6. **Active filter chips** -- horizontal scroll row
7. **Sort pills + Results count** -- row
8. **Results** -- single column, full-width cards
9. **Load More** -- full-width button

### Filter Panel Layout (Claude's Discretion Decision)

**Desktop: Left sidebar (280px fixed width).** Rationale: Neo-brutalism's bold borders and high-contrast panels work naturally as a persistent sidebar. A sidebar keeps filters always visible without hiding them behind a toggle, reducing interaction cost for the power-user pattern of iterative filter refinement. The 280px width accommodates multi-select chips and searchable dropdowns without cramming.

**Mobile: Bottom sheet** triggered by a "Filters" button. The smart eligibility bar stays at top. The bottom sheet contains all other filters (degree, field of study, funding, prestige). Sheet slides up to 85% viewport height with internal scroll.

### Featured Row Layout (Claude's Discretion Decision)

**Mobile: Horizontal scroll** with snap points. Rationale: Horizontal scroll preserves vertical real estate on mobile while allowing browse-through of 5-6 featured cards. Cards snap to center alignment. No vertical stack -- that would push the main directory too far down.

### Grid Layout

| Breakpoint | Columns | Card Width | Gap |
|------------|---------|------------|-----|
| < 640px (mobile) | 1 | 100% | 16px |
| 640-1023px (tablet) | 2 | 1fr | 24px |

- Fixed height: 320px (desktop/tablet), auto on mobile
- Content truncation: title 2 lines max, description 2 lines max

## Interaction Contract

### Smart Eligibility Filter Bar

| State | Visual | Behavior |
|-------|--------|----------|
| Empty (first visit) | Two-line layout: "I'm from [Select nationality v] looking to study in [Select destinations v]" | Country selectors open on click. Popular chips shown below destination selector. |
| Populated | Full bar showing selections: flag emoji + country names | Saves to localStorage AND URL params |
| Compact (after set) | Single line: "Flag Bangladesh -> Flag DE Flag GB Flag US [Edit]" | Click "Edit" expands back to full bar |
| Nationality detected | Soft yellow banner above bar: "It looks like you might be in Bangladesh. [Set as nationality]" | Clicking sets nationality, dismisses banner. Banner dismissible with X. localStorage flag prevents re-show. |

### Search Suggestions

| State | Visual | Behavior |
|-------|--------|----------|
| Typing (>= 2 chars) | Radix Popover dropdown below search input, max 5 items | Shows mixed: scholarship title matches (with host country flag) + filter category suggestions (e.g., "Engineering" as field of study) |
| No suggestions | Dropdown hidden | -- |
| Enter pressed | Dropdown closes, full results update | Adds `q=` to URL params |
| Suggestion clicked | Dropdown closes, navigates to scholarship OR applies filter | Title click -> detail page; category click -> applies filter |

### Filter Interactions

| Filter | Control Type | Behavior |
|--------|-------------|----------|
| Degree level | Multi-select chips (Bachelor, Master's, PhD, Postdoc) | Toggle on/off. Active chips get accent border + filled background. |
| Field of study | Searchable multi-select dropdown (Radix Select) | Type to filter ~25 categories. Selected items shown as removable pills. |
| Funding type | Multi-select chips | Same as degree level |
| Prestige tier | Multi-select chips (Gold, Silver, Bronze) | Chips colored with their tier color |
| Show closed | Toggle switch | Off by default. When on, closed scholarships appear at bottom with grey overlay |
| Show ineligible | Toggle switch | Below eligibility bar. "Also show scholarships you may not qualify for" |

### Quick Filter Tabs

| Tab | Behavior | URL |
|-----|----------|-----|
| Open Now | Filter: status=published, deadline > now | Default active |
| Closing Soon | Filter: deadline < 30 days from now | `/scholarships?closing_soon=true` or `/scholarships/closing-soon` |
| Fully Funded | Filter: funding_type=fully_funded | Combinable with others |
| Gold Tier | Filter: prestige_tier=gold | Combinable with others |

Tabs use AND logic when multiple are active. Visually: active tabs get accent background + white text + neo-brutalism shadow.

### Sort Pills

| Pill | Sort Logic | Default Direction |
|------|-----------|-------------------|
| Deadline | `application_deadline` ASC | Soonest first (default active) |
| Prestige | `prestige_tier` DESC then `prestige_score` DESC | Gold first |
| Newest | `_creationTime` DESC | Most recent first |
| Amount | `award_amount_max` DESC | Highest first |

Active pill: accent background, white text, shadow. Inactive: neutral variant.

### Pagination (Claude's Discretion Decision)

**"Load More" button** at bottom of results. Initial load: 20 scholarships. Each "Load More" appends 20 more. Rationale: Infinite scroll is problematic for SEO and makes it hard to reach the footer/navigate away. Traditional pagination (page 1, 2, 3...) doesn't work well with Convex's cursor-based pagination model. "Load More" preserves scroll position, works naturally with Convex `usePaginatedQuery`, and stays within free-tier budget by loading on demand rather than eagerly.

### Card Interactions

| Element | Interaction | Visual Feedback |
|---------|------------|-----------------|
| Card (entire) | Click -> scholarship detail page | Card translates by shadow offset on hover (existing neo-brutalism pattern) |
| "Copy Link" button | Click -> copies card URL to clipboard | Button text changes to "Copied!" for 2 seconds |
| Host country flag badge | Display only | Positioned top-right of card |
| Prestige badge | Display only + tooltip on hover | Tooltip explains tier: "Gold: Fully funded by a flagship provider, hosted in a top-tier academic country" |
| Deadline badge | Display only | Color-coded per urgency table |
| "New" badge | Display only | Accent color with subtle pulse animation (2s cycle, opacity 0.7 -> 1.0) |
| "Limited Info" badge | Display only | Neutral variant, indicates incomplete data |

### Loading States

| State | Visual |
|-------|--------|
| Initial load | 6 skeleton cards in grid (or 3 skeleton rows in list view) matching card dimensions. Animated shimmer effect. |
| Load more | 3 skeleton cards appended below existing results |
| Filter change | Existing results remain visible with 50% opacity while new results load. No skeleton flash for quick updates. |
| Error | Inline error banner above results: red-bordered card with error icon + message + "Retry" button |

### Dark Mode Toggle

| State | Visual | Location |
|-------|--------|----------|
| Light mode | Sun icon (Lucide `Sun`) | Navbar, right side |
| Dark mode | Moon icon (Lucide `Moon`) | Navbar, right side |
| Toggle | Instant class toggle on `<html>`, persisted to localStorage | Prevents FOUC via `ScriptOnce` blocking script |

## Empty State Illustration

**Style (Claude's Discretion Decision):** Neo-brutalism geometric illustration. A bold-outlined magnifying glass tilted at 15 degrees with 3 geometric shapes (circle, triangle, square) floating around it, rendered with 2px black strokes, filled with the accent color at 20% opacity. No gradients, no soft shadows, no rounded-cute aesthetic. The illustration should feel like a bold poster graphic, not a friendly mascot. Maximum 120x120px SVG, inline in the component.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| WCAG AA contrast (4.5:1 text) | All text on prestige-tinted backgrounds verified against OKLCH values. Gold/Bronze backgrounds are light enough for black text. Silver background uses black text. Dark mode badges use high-lightness text. |
| WCAG AA contrast (3:1 UI) | All borders, icons, and interactive element boundaries meet 3:1 against their backgrounds |
| Keyboard navigation | All filter chips, sort pills, quick filters, cards focusable via Tab. Enter/Space activates. Radix primitives handle this natively. |
| Screen reader labels | `aria-label` on filter chips ("Remove PhD filter"), sort pills ("Sort by deadline, currently active"), view toggle ("Switch to list view"), search input ("Search scholarships"), country selectors |
| Focus indicators | 2px solid `var(--ring)` outline with 2px offset on all interactive elements |
| Touch targets | Minimum 44px height on all tappable elements |
| RTL-ready layout | Use CSS logical properties: `padding-inline-start` not `padding-left`, `margin-inline-end` not `margin-right`, `inset-inline-start` not `left`. Flexbox `gap` instead of directional margins where possible. |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables "New" badge pulse animation and card hover translate transitions |

## URL State Contract

All filter state is encoded in URL query parameters for shareability and SSR meta tag generation.

| Parameter | Format | Example |
|-----------|--------|---------|
| `q` | Free text | `q=DAAD` |
| `from` | Comma-separated ISO 3166-1 alpha-2 | `from=BD,IN` |
| `to` | Comma-separated ISO 3166-1 alpha-2 | `to=DE,GB,US` |
| `degree` | Comma-separated enum values | `degree=phd,masters` |
| `field` | Comma-separated slug values | `field=engineering,medicine` |
| `funding` | Comma-separated enum values | `funding=fully_funded` |
| `tier` | Comma-separated enum values | `tier=gold,silver` |
| `sort` | Single enum value | `sort=prestige` |
| `view` | Single enum value | `view=list` |
| `show_closed` | Boolean | `show_closed=true` |
| `show_ineligible` | Boolean | `show_ineligible=true` |

Full example: `/scholarships?from=BD&to=DE,GB,US&degree=phd,masters&field=engineering&funding=fully_funded&tier=gold&q=DAAD&sort=deadline&view=grid`

Managed via TanStack Router `validateSearch` with Zod schema. Nationality (`from`) additionally persisted to localStorage for returning visitors.
