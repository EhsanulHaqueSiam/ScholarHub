1. **Create `web/src/routes/scholarships/index.tsx`** -- Main directory page:

This is the primary page of the application. Uses `createFileRoute("/scholarships/")`.

**Route definition:**
```typescript
import { createFileRoute } from "@tanstack/react-router";
import { scholarshipSearchSchema } from "@/lib/filters";

export const Route = createFileRoute("/scholarships/")({
  validateSearch: scholarshipSearchSchema,
  head: ({ search }) => {
    const parts: string[] = [];
    if (search.degree) parts.push(formatDegree(search.degree));
    parts.push("Scholarships");
    if (search.to) parts.push(`in ${formatCountries(search.to)}`);
    if (search.from) parts.push(`for ${formatNationalities(search.from)} Students`);
    const title = parts.join(" ") + " -- ScholarHub";
    const description = `Browse international scholarships${search.to ? ` in ${formatCountries(search.to)}` : ""}. Filter by degree, funding, and eligibility.`;

    return {
      meta: [
        { title },
        { name: "description", content: description },
        { property: "og:title", content: title },
        { property: "og:description", content: description },
        { property: "og:type", content: "website" },
      ],
    };
  },
  component: ScholarshipsDirectory,
});
```

Helper functions `formatDegree`, `formatCountries`, `formatNationalities` convert comma-separated codes to human-readable text. Use `getCountryName` from countries.ts.

3. **Create `web/src/routes/scholarships/$slug.tsx`** -- Detail page placeholder with Schema.org JSON-LD:

Per CONTEXT.md locked decision: "Basic Schema.org structured data (Scholarship JSON-LD) on each scholarship page"

Add a `getBySlug` query to `web/convex/directory.ts`:
```typescript
export const getBySlug = query({
  args: { slug: v.string() },
  handler: async (ctx, { slug }) => {
    return await ctx.db
      .query("scholarships")
      .withIndex("by_slug", (q) => q.eq("slug", slug))
      .first();
  },
});
```
If the `by_slug` index doesn't exist, add `.index("by_slug", ["slug"])` to the scholarships table in schema.ts.

The route component queries the scholarship by slug and embeds Schema.org JSON-LD structured data using a `<script type="application/ld+json">` tag. The JSON-LD object uses `@type: "Scholarship"` with fields: name, description, provider (Organization), url, applicationDeadline, eligibleRegion (array of Place), studyLocation (Place), amount (MonetaryAmount). All data comes from the Convex database (trusted source, not user-generated content -- safe for inline script).

Also renders a minimal placeholder UI: Navbar, scholarship title, provider, and "Full detail page coming in Phase 7" message.
