## Decisions Made
- Dual query path: search index for text search, compound indexes for non-search listing -- Convex search indexes cannot be combined with sort orders, so non-search queries use by_status_deadline or by_status_prestige_deadline indexes
- Multi-select funding type handled via single-value push to search index (when exactly 1 selected) or post-filter (when 2+ selected) -- Convex search index filterFields only support .eq() (single value)
- Manual cursor-based pagination for search results since Convex search index returns all matching results via .collect() -- standard .paginate() only works with index-based queries
- Nationality eligibility implemented as post-filter (after paginate) because Convex .filter() expressions cannot do array.includes() on eligibility_nationalities
