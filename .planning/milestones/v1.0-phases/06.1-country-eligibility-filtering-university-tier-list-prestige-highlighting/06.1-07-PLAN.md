**1. Accessibility (WCAG AA)**

Add to every interactive element across all listed files:
- `aria-label` descriptors on EVERY interactive element:
  - SearchBar: `aria-label="Search scholarships"` on input
  - CountrySelector: `aria-label` with placeholder text on trigger button, `aria-label="Search countries"` on search input
  - EligibilityFilterBar: `aria-label="Set nationality"`, `aria-label="Set study destinations"` on selectors, `aria-label="Toggle show ineligible scholarships"` on toggle
  - NationalityBanner: `aria-label="Set as nationality"` on CTA button, `aria-label="Dismiss nationality suggestion"` on X
  - FilterPanel: `aria-label="Filter by degree level"`, `aria-label="Filter by field of study"`, `aria-label="Filter by funding type"`, `aria-label="Filter by prestige tier"`, `aria-label="Toggle show closed scholarships"`
  - FilterChips: `aria-label="Remove [filter name] filter"` on each chip remove button, `aria-label="Clear all filters"` on clear button
  - QuickFilters: `aria-label="Filter: Open Now"`, `aria-label="Filter: Closing Soon"`, `aria-label="Filter: Fully Funded"`, `aria-label="Filter: Gold Tier"` on each toggle
  - SortPills: `aria-label="Sort by deadline"`, `aria-label="Sort by prestige"`, `aria-label="Sort by newest"`, `aria-label="Sort by amount"` on each pill
  - ViewToggle: `aria-label="Switch to grid view"`, `aria-label="Switch to list view"` on icons
  - ScholarshipCard: `aria-label` with scholarship title on card link, `aria-label="Copy link to [title]"` on copy button
  - ScholarshipListItem: same as ScholarshipCard
  - FeaturedRow: `aria-label="Scroll left"`, `aria-label="Scroll right"` on arrow buttons
  - Navbar: `aria-label="Toggle dark mode"` on toggle, `aria-label="Main navigation"` on nav element
  - BackToTop: `aria-label="Back to top"` on button
  - EmptyState: `role="status"` on container

- `aria-pressed` on toggle chips/buttons (quick filters, sort pills, view toggle)
- `aria-expanded` on expandable elements (eligibility bar compact/expanded, mobile filter button, search dropdown)
- `role="combobox"` on SearchBar, `role="listbox"` on suggestion dropdown, `role="option"` on each suggestion
- `aria-live="polite"` on results count element (so screen readers announce count changes)
- Focus indicators: ensure `focus-visible:outline-2 focus-visible:outline-ring focus-visible:outline-offset-2` on all interactive elements
- Tab order: logical flow through page
- Skip link: `<a href="#results" className="sr-only focus:not-sr-only focus:absolute focus:top-2 focus:left-2 focus:z-50 bg-main text-main-foreground px-4 py-2 rounded-base">Skip to results</a>` at top of directory page. Add `id="results"` to results area.

Touch targets: verify all tappable elements are min 44px height. Add `min-h-[44px]` where needed.

**2. RTL-Ready Layout Audit**

Search all component files and replace directional CSS with logical equivalents:
- `pl-*` -> `ps-*`, `pr-*` -> `pe-*`, `ml-*` -> `ms-*`, `mr-*` -> `me-*`
- `left-*` -> `start-*`, `right-*` -> `end-*`
- `text-left` -> `text-start`, `text-right` -> `text-end`
- Use `gap` instead of directional margins where possible
- Exception: `translate-x-*` for hover effects stays as-is

**3. Reduced Motion**

Add `@media (prefers-reduced-motion: reduce)` handling:
- ScholarshipCard: `motion-safe:hover:translate-x-boxShadowX`
- Badge "New" pulse: `motion-safe:animate-pulse`
- SkeletonCard: `motion-safe:animate-pulse`
- BackToTop: `motion-safe:transition-opacity`
- Scroll behavior: `behavior: "auto"` when `prefers-reduced-motion: reduce`

**4. Copy Link Button on Cards**

In ScholarshipCard and ScholarshipListItem, implement the "Copy Link" button:
- On click: copy `/scholarships/${scholarship.slug}` to clipboard
- After copy: text changes to "Copied!" for 2 seconds
- `aria-label="Copy link to ${scholarship.title}"`
- Stop propagation so click doesn't navigate

**5. Error Handling**

Create `web/src/components/ErrorBoundary.tsx` -- class-based React error boundary.
Create `DirectoryErrorFallback` in directory route with "Unable to load scholarships" + Retry button.
Wrap results section in ErrorBoundary.

**6. Dark Mode Verification**

Ensure all components use CSS variable-based colors (not hardcoded):
- Prestige tints: `bg-prestige-gold-bg` not hardcoded oklch
- Urgency badges: `bg-urgency-critical` not hardcoded
- Text: `text-foreground` not `text-black`
- Verify dark mode toggle correctly toggles `dark` class on `<html>`

**7. Prestige Tooltip on Badges**

Add title/tooltip to prestige badges using `getPrestigeTooltip` from `@/lib/prestige.ts`.
