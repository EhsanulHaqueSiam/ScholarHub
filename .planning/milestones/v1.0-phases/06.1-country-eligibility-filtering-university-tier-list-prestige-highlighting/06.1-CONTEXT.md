<decisions>
## Implementation Decisions

### Smart Eligibility Filter Bar (top prominence)
- Combined smart filter: "I'm from [nationality] looking to study in [destinations]" — positioned at top prominence above all other filters
- **Auto-detect nationality** via browser signals (Claude picks best approach: locale, timezone, or IP) with soft suggestion banner: "It looks like you might be in [Country]. Set your nationality to see eligible scholarships." — never auto-filter without explicit action
- **Multi-nationality support** (up to 3) — handles dual citizens and PR holders
- **Multi-destination support** — select multiple host countries with AND logic
- **Popular destinations** as quick-pick chips (top 15 study-abroad destinations, static curated list initially, switch to data-driven later)
- Country display: flag emoji + full name (e.g., 🇧🇩 Bangladesh)
- Country list organized: popular nationalities first (~15-20 most common), then searchable full list below
- Scholarships with no nationality restriction ("open to all") **always shown** regardless of eligibility filter
- **"Also show scholarships you may not qualify for"** toggle positioned below the smart filter bar
- **Compacts after set**: after setting nationality + destinations, bar compacts to a single line: "🇧🇩 Bangladesh → 🇩🇪🇬🇧🇺🇸 [Edit]"
- **Persistence**: localStorage for returning visitors AND encoded in URL for sharing
- **URL format**: `/scholarships?from=BD,IN&to=DE,GB,US` — ISO country codes, query params
- Host country flag badge on each scholarship card showing which destination it matches

### Prestige Tier System (Gold / Silver / Bronze)
- **Combined prestige score** — composite of scholarship competitiveness + provider reputation + funding level + host country academic rank
- **3 tiers**: Gold (elite), Silver (strong), Bronze (standard), plus Unranked (insufficient data)
- **Score factors** (all four):
  1. Funding level — fully funded > partial > tuition waiver > stipend only
  2. Provider reputation — hardcoded seed list of ~50-75 well-known prestigious providers (Fulbright, Rhodes, DAAD, Chevening, Erasmus Mundus, Gates Cambridge, MEXT, CSC, etc.), admin-editable later
  3. Competitiveness indicators — acceptance rate, applicant volume, "highly competitive" tags from scraped data (optional — score works without it)
  4. Host country academic rank — static 3-tier list: Tier A (USA, UK, Germany, Japan, etc.), Tier B (moderate), Tier C (developing)
- **Rule-based auto-assignment** with admin override capability
- **Auto-recalculate** whenever scholarship data changes (Convex mutation, write-time calculation)
- **Admin-configurable weights** stored in Convex — e.g., funding_level=40%, provider_reputation=30%, country_rank=20%, competitiveness=10%
- **Visible + sortable** — students see tier badges and can sort/filter by tier
- **Unranked** (missing prestige data) = standard card, no badge — distinct from Bronze which has a bronze badge
- Prestigious providers: hardcoded seed list in codebase, admin can add/remove via dashboard later
- Country academic tiers: hardcoded constants in codebase

### Prestige Visual Highlighting
- **Full card color tint** — entire card background gets gold/silver/bronze tint
- **All 3 tiers styled** differently with color-coded tints
- **Bold and obvious** tints — strong, immediately noticeable, fits neo-brutalism's high-contrast philosophy
- Unranked scholarships: standard card, no badge, no tint
- **Detail page**: tier badge at top with explanation tooltip: "Gold: Fully funded by a flagship provider, hosted in a top-tier academic country"
- **Featured "Top Scholarships" row** at the top of directory:
  - 5-6 cards, static with scroll arrows (no auto-carousel)
  - Personalized by nationality + sorted by deadline urgency
  - Fill with Silver if not enough Gold scholarships
- **Dark mode**: launch with both light and dark mode. Dark mode tint design at Claude's discretion
- Tint contrast vs accessibility: Claude balances bold tints with WCAG AA requirements

### Search Experience
- **Hybrid: suggestions + submit** — top 5 suggestions as dropdown while typing, full results on Enter
- Suggestions show **mixed: scholarship titles + filter category suggestions** (e.g., "Engineering" as field of study filter)
- Search fields: **title + description + eligibility text**
- **Search within filtered results** — AND logic with active filters
- Search bar positioned **above all filter panels** (top of page hierarchy: Search → Smart Eligibility → Other Filters → Results)
- **Zero results**: illustration (neo-brutalism styled) + guidance ("Try removing [most restrictive filter]") + "Did you mean...?" suggestions + popular scholarships fallback
- Search query **persists in URL**: `/scholarships?q=engineering&from=BD&to=DE`
- Convex search index on title with filterFields for status/country/funding_type (already exists — may need extension for eligibility text)

### Listing Cards
- Cards show **all info groups**: title, host country (flag badge), deadline (color-coded), funding type + amount, degree levels + field of study, provider name, prestige tier badge, 1-2 line truncated description snippet
- **Switchable grid (3-col) + list view** — toggle between views
- Grid: **fixed height, truncate** — all cards same height, long content truncated with ellipsis
- List view: **more detail** — additional fields including 3-4 line description, eligibility text, provider details
- **Deadline urgency color coding**: Red (<7 days), Yellow/Amber (<30 days), Green (open), Grey (closed)
- **"New" badge** for scholarships published in the last 7 days
- **Results count prominently displayed**: "47 scholarships for 🇧🇩 Bangladesh in 🇩🇪 Germany"
- Total scholarship count as trust signal: "Browse 2,400+ International Scholarships" headline
- Missing data: show what exists, skip missing fields. Very incomplete scholarships (title + URL only) shown with "limited info" badge

### Filter Panel
- Filter position: Claude's discretion (sidebar vs top bar) based on neo-brutalism design and mobile responsiveness
- **Active filter chips** always shown above results as removable pills: "Degree: PhD × | Funding: Fully Funded × | Clear all"
- **Quick filter tabs** (combinable toggle chips): "Open Now" | "Closing Soon" | "Fully Funded" | "Gold Tier" — AND logic when multiple active
- Degree level: **multi-select chips** (Bachelor | Master's | PhD | Postdoc)
- Field of study: **searchable multi-select** (20-30 broad categories)
- Funding type: **multi-select chips** (Fully Funded | Partial | Tuition Waiver | Stipend Only)
- Prestige tier: **sort + filter both** available
- **Full URL state**: `/scholarships?from=BD&to=DE&degree=phd,masters&field=engineering&funding=fully_funded&tier=gold&q=DAAD`

### Sort Options
- 4 sort options: Deadline (soonest first, default), Prestige tier (Gold first), Recently added, Funding amount (highest first)
- Sort UI: **pill buttons** — Deadline | Prestige | Newest | Amount
- **Smart defaults only** — each sort has one sensible direction, no toggle

### Closing Soon & Expired Scholarships
- **"Closing Soon"**: quick filter chip + dedicated URL (`/scholarships/closing-soon`) for bookmarking/SEO
- Closing Soon and prestige are **independent** — urgency view shows all tiers, sorted by deadline
- **Expired scholarships hidden by default**, toggle: "Show closed scholarships"
- When shown, expired scholarships display **expected reopen date** when available: "Applications Closed · Expected to reopen: September 2026"

### Pagination & Loading
- Pagination approach: Claude's discretion based on data volume (~2000-5000) and Convex free tier optimization
- **Skeleton cards** as loading state while fetching from Convex
- **Live updates** via Convex reactive queries — new scholarships appear immediately when admin publishes

### Navigation & Page Structure
- **Navbar**: ScholarHub logo + "Scholarships" (directory) + "Closing Soon" (urgent view)
- **Full hero section** with bold neo-brutalism styling: headline, description, prominent search bar. Directory below the fold
- **Floating back-to-top button** appears after scrolling past first page of results
- Route structure set up for Phase 9: `/scholarships` (directory), `/scholarships/[country]` (country landing), `/scholarships/[degree]` (degree landing)

### Mobile
- Cards: **full-width single column**
- Filters: **bottom sheet** — tap "Filters" button → bottom sheet slides up. Smart filter bar stays visible at top
- Featured row: Claude's discretion (horizontal scroll vs vertical stack)

### Performance & Convex
- **Server-side filtering** — Convex query applies all filters, returns only matching results. Uses compound indexes
- Prestige score calculated at **write time** (Convex mutation) and stored on scholarship record
- Expected data volume: ~2000-5000 scholarships
- Query strategy within Convex free tier (1M calls/month)

### Schema Changes
- Add `prestige_tier` (gold/silver/bronze/unranked) and `prestige_score` (number) fields to scholarships table
- Add appropriate indexes for prestige-based queries
- Convex search index may need extension for eligibility text search

### SEO Preparation
- **Migrate to TanStack Start** (SSR) from TanStack Router — included as first task in this phase
- **Basic Schema.org structured data** (Scholarship JSON-LD) on each scholarship page
- **Dynamic meta tags** based on filter state: "PhD Scholarships in Germany for Bangladeshi Students — ScholarHub"
- **Dynamic OG tags** via SSR for rich social previews when sharing filtered URLs
- **Route placeholders** for Phase 9 landing pages created now

### Share & Social
- **Copy link button** on cards and filtered views — copies URL with full filter state
- **Dynamic OG tags** via TanStack Start SSR — shared URLs generate rich previews

### Accessibility & i18n
- **WCAG AA** accessibility: keyboard navigation, screen reader labels, 4.5:1 contrast ratios, focus indicators
- **RTL-ready layout** — build with logical CSS properties (start/end not left/right) for future RTL support
- English-only UI for v1. Scholarship content in other languages displayed as-is

### Dark Mode
- **Launch with both** light and dark mode from the start (CSS variables already defined)
- Dark mode prestige tints: Claude's discretion to design appropriate dark mode adaptation

### Data Completeness
- Missing fields: **show what exists, skip missing** — cards vary in content based on available data
- Very incomplete scholarships (title + URL only): **shown with "limited info" badge**

### Error Handling
- **Inline error + retry** for Convex/network failures: "Unable to load scholarships. Check your connection and try again." with Retry button

<specifics>
## Specific Ideas

- Smart filter bar should feel like a personal assistant: "I'm from Bangladesh, looking to study in Germany, UK, USA" — conversational, not form-like
- Prestige tints should be bold and unmissable — this is neo-brutalism, not Material Design. Gold should feel premium, Silver should feel solid, Bronze should feel warm
- "Browse 2,400+ International Scholarships" as trust signal headline — students should feel the catalog depth immediately
- Featured Top Scholarships row is personalized: if a Bangladeshi student visits, they see Gold-tier scholarships they qualify for, sorted by deadline urgency
- Copy link is the share mechanism — WhatsApp is how students share in many markets, and paste-a-link works everywhere
- Illustration in empty states should be neo-brutalism: bold lines, high contrast, maybe geometric/abstract — not cute/cartoon

</specifics>
