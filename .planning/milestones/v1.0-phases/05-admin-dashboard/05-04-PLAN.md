Editor setup:
```typescript
const editor = useEditor({
  extensions: [
    StarterKit.configure({
      heading: { levels: [2, 3] },
    }),
    Link.configure({ openOnClick: false, HTMLAttributes: { class: "text-main underline" } }),

  onUpdate: ({ editor }) => onChange(editor.getHTML()),
```

Per RESEARCH Pitfall 5: TipTap stores HTML, but existing auto-generated notes from `demoteIncompleteScholarships` are plain text/markdown. Need dual-format rendering.

Read the current component. It uses react-markdown. Modify to detect content format:

```typescript
// Detection: if content starts with < or contains common HTML tags, treat as HTML
const isHtml = (content: string) => /^<[a-z]/i.test(content.trim()) || /<\/?(?:p|div|ul|ol|li|h[1-6]|strong|em|a|br)\b/i.test(content);
```

For HTML content: Use a sanitizer approach. Install DOMPurify (`npm install dompurify` + `npm install -D @types/dompurify`) and sanitize before rendering:

```typescript
import DOMPurify from "dompurify";

if (isHtml(notes)) {
  const sanitized = DOMPurify.sanitize(notes, {
    ALLOWED_TAGS: ["p", "strong", "em", "a", "ul", "ol", "li", "h2", "h3", "br"],
    ALLOWED_ATTR: ["href", "target", "rel", "class"],
  });
  return (
    <div className="prose prose-sm max-w-none"
         // Using sanitized HTML from admin-controlled TipTap input
         // Sanitized with DOMPurify whitelist to prevent XSS
         dangerouslySetInnerHTML={{ __html: sanitized }} />
  );
}
// Existing react-markdown rendering for markdown content
```

This is defense-in-depth: even though HTML comes from admin-controlled TipTap input, DOMPurify strips any unexpected tags/attributes as a safety measure.

For markdown content: continue using react-markdown (existing behavior).
