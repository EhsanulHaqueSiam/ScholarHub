4. **Create SourceConfig Protocol** (`_protocol.py`):
   ```python
   from typing import Protocol, runtime_checkable

   @runtime_checkable
   class SourceConfig(Protocol):
       """Protocol all source configs must implement."""
       name: str                         # Human-readable name matching catalog
       url: str                          # Base URL of the source
       source_id: str                    # Matches catalog entry name (slug form)
       primary_method: str               # api, jsonld, ajax, rss, scrape, scrapling
       secondary_method: str | None      # Fallback method (explicit per source)
       selectors: dict[str, str]         # CSS/XPath selectors for data extraction
       field_mappings: dict[str, str]    # Maps extracted fields to raw_record schema
       pagination: dict | None           # {"type": "url"|"cursor"|"page_num", "selector": "...", "max_pages": N}
       detail_page: bool                 # Whether to follow links for full data
       detail_selectors: dict[str, str] | None  # Selectors for detail pages
       auth_config: dict | None          # Future: auth credentials (None in v1)
       rate_limit_delay: float           # Seconds between requests (per source)
       cutoff_months: int                # Stop paginating after this many months expired (default 3)
   ```

   ```python
   import importlib
   import pkgutil
   from pathlib import Path

   def discover_configs() -> list:
       """Scan configs directory, import all modules implementing SourceConfig."""
       configs = []
       package_dir = Path(__file__).parent
       for importer, modname, ispkg in pkgutil.iter_modules([str(package_dir)]):
           if modname.startswith("_"):
               continue
           module = importlib.import_module(f".{modname}", package=__name__)
           # Look for a CONFIG attribute (convention: each config module exports CONFIG)
           if hasattr(module, "CONFIG"):
               configs.append(module.CONFIG)
       return configs
   ```

2. **Create web/convex/scraping.ts** with internalMutation functions:

   - `startRun`: Creates a new scrape_runs record with status "running", returns the run `_id`. Args: `{ triggered_by: v.string(), sources_targeted: v.number() }`.
   - `completeRun`: Patches scrape_runs record with final stats and status. Args: `{ run_id: v.id("scrape_runs"), status, sources_completed, sources_failed, records_inserted, records_updated, records_unchanged, duration_seconds }`.
   - `batchInsertRawRecords`: Accepts array of record objects + run_id. For each record: query by_source_external index to find existing. If existing, compute field-level diff (compare title, description, host_country, application_deadline, award_amount, source_url), insert change_log entries for changed fields, then patch the existing record. If new, insert with scraped_at=Date.now(). Returns `{ inserted: number, updated: number, unchanged: number }`.
   - `recordSourceResult`: Inserts a scrape_run_sources telemetry entry. Args match the table schema.
   - `updateSourceHealth`: Upserts source_health record by source_id. On success: reset consecutive_failures to 0, set status based on yield vs avg_yield (healthy if yield >= 50% avg, degraded otherwise), update last_success, last_yield, recalculate avg_yield as rolling average. On failure: increment consecutive_failures, set status to "failing" if >= 5, update last_failure + error info.
   - `updateLastScraped`: Patches sources table `last_scraped` field for a given source_id. This fulfills SCRP-07 (last verified timestamp).

   All functions use `internalMutation` (not public mutation) -- they are called via admin auth from Python SDK.

3. **Create web/convex/maintenance.ts** with internal mutations:
   - `cleanupOldRecords`: Delete raw_records where `scraped_at < Date.now() - 90 * 24 * 60 * 60 * 1000`. Process in batches of 100 to avoid timeout.
   - `cleanupChangeLog`: Delete change_log where `changed_at < Date.now() - 90 * 24 * 60 * 60 * 1000`. Batch of 100.

4. **Create web/convex/monitoring.ts** with internal mutations/queries:
   - `checkHeartbeat`: Query latest scrape_runs record. If `started_at` is older than 48 hours and no run has status "completed" in last 48h, log a warning (Convex cron can't create GitHub Issues, but this flags in Convex logs).
   - `getStaleHeartbeat`: Internal query returning boolean -- whether heartbeat is stale.

5. **Create web/convex/dashboard.ts** with public queries (pre-built for Phase 5):
   - `getRecentRuns`: Query scrape_runs ordered by started_at desc, limit 10. Return with computed duration.
   - `getSourceHealth`: Query all source_health records, join with sources table for name. Return sorted by status (failing first).
   - `getFailingSources`: Query source_health with status "failing" or "deactivated".
   - `getRunStats`: Query single scrape_run by ID, plus all scrape_run_sources for that run.

6. **Create web/convex/crons.ts**:
